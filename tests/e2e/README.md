# E2E Testing — Webwright + Playwright

This directory contains a hybrid E2E framework:

- **One-time local login** captures OAuth cookies via Playwright.
- **Webwright** (LLM-driven browser agent) generates stable Python/Playwright test scripts.
- **pytest** runs accumulated scripts in CI with zero LLM calls.

---

## Directory layout

```
tests/e2e/
├── auth/                    # Local only — auth_state.json lives here (git-ignored)
├── generated_scripts/       # Committed Playwright tests generated by Webwright
└── templates/
    └── task_template.json   # Fill-in template for dispatching Webwright tasks
```

---

## Step 1 — Capture login state (one-time, local only)

Make sure `pnpm dev` is running, then:

```bash
pnpm test:e2e:login
```

A browser window opens. Complete the Google OAuth + 2FA flow manually, then close it.
The session is saved to `tests/e2e/auth/auth_state.json` (git-ignored — never commit).

---

## Step 2 — Generate a test with Webwright

Install Webwright (see [microsoft/webwright](https://github.com/microsoft/webwright)):

```bash
pip install webwright   # or follow repo instructions
```

With `pnpm dev` running, invoke the CLI:

```bash
python -m webwright.run.cli \
  -c src/webwright/config/base.yaml \
  -c src/webwright/config/model_claude.yaml \
  --start-url "http://localhost:3000/" \
  -t "載入 tests/e2e/auth/auth_state.json 的登入狀態。進入首頁後，輸入謠言「喝冰水會中風」，點擊發送，靈活等待最多 30 秒直到 data-testid='assistant-message' 出現且不再顯示 loading，確認畫面沒有崩潰。" \
  --task-id "test_chat_stream" \
  -o tests/e2e/generated_scripts/
```

### cofacts/ai routing & streaming hints for Webwright prompts

| Concern | Detail |
|---|---|
| Routing | TanStack Start file-based routes: `/chat`, `/profile`, `/settings` |
| Streaming signal | ADK response done when `[data-testid="assistant-message"]` loses its spinner/loading class |
| API backend | FastAPI at `http://localhost:8000`; chat uses SSE streaming |
| Auth cookie | Pre-loaded from `auth_state.json` — no re-login needed inside the task prompt |

Use `tests/e2e/templates/task_template.json` as a fill-in guide for new task prompts.

---

## Step 3 — Review, rename, and commit

Webwright writes output under `tests/e2e/generated_scripts/<task_id>/`.  
Review the script, rename/clean it (e.g. `test_chat_stream.py`), then commit.

---

## Step 4 — Run all tests (CI / local)

```bash
pnpm test:e2e:run-all
# equivalent to: pytest tests/e2e/generated_scripts/
```

No LLM API is called at runtime — pure Playwright execution.
